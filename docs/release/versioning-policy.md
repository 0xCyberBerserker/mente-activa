# Versioning Policy

## Espanol

- `VERSION.json` solo se cambia al preparar una release.
- Si no existe una release publica valida, `0.0.1-alpha` puede ser la primera alpha presentable.
- Si ya existe una release publica valida, la version nunca se reduce.
- Las alpha avanzan de forma secuencial como `0.0.1`, `0.0.2`, `0.0.3`.
- Los commits ordinarios no cambian la version.

## English

- `VERSION.json` changes only when preparing a release.
- If no valid public release exists, `0.0.1-alpha` may be the first presentable alpha.
- If a valid public release already exists, the version must never be reduced.
- Alpha versions advance sequentially as `0.0.1`, `0.0.2`, `0.0.3`.
- Ordinary commits do not change the version.
